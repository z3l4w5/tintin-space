    2.3 添加内容 #

    我们将转换与Eclectic主题产生的空页面为一个功能齐全的网站。 这种转换包括通过为主题提供一些设置和元数据来配置主题，添加像隐私政策和使用条款这样的页面， 并使用自定义版本覆盖主题的登录页面。

    注 主题的自定义数据不可移植。 你将不得不查看主题的文档以找出主题特定的配置。 如果你在开发网站时仍在判断主题，建议你首先关注基于模板的标准内容页面(如帖子)，而不是独特的页面(如登录页面和联系我们)。

    2.3.1 配置 #

    网站用两行代码运行得如此之好，这就是Hugo经过深思熟虑的默认设置的魔力所在。 我们可以通过为我们的网站传递正确的选项来做得更好。 配置文件有两个截然不同的部分：顶层配置和特定于主题的参数部分，前者在主题中是通用的，后者在不同的主题中有所不同。 让我们在配置文件config.yaml中添加一些数据。 这是HugoEclectic模板取得成功所必需的。 根据Acme CorporationEclectic主题的要求，这些更改提供了填充菜单，页脚，版权声明以及标题和作者信息的信息。 更新的配置文件位于本书( https://github.com/hugoinaction/hugoinaction/tree/第2章的资源文件夹中(第02章-资源/02)。 你还不需要了解整个文件。 我们将在接下来的章节中使用这些设置，这些设置将变得更加清晰。 清单2.5显示了我们将用于Acme Corporation网站的配置文件。 典型的Hugo配置文件包含：

    • 所有主题的标准配置选项 (例如网站的URL，其名称和语言)
    • 针对特定Hugo功能的选项(如菜单)
    • 主题特定参数 (如参数)
    Listing 2.5 Setting up Acme Corporation’s configuration file (config.yaml)

    Hugo通过称为taxonomies的功能支持多位作者 (在第4.4节中讨论)。 Hugo还提供了定义菜单的标准方法。 配置文件中的菜单部分具有键，每个键指定一个菜单名称。 每个菜单都有一个条目列表，这些条目可以具有唯一的标识符、要显示的名称、URL和用于对菜单项进行排序的权重。 在配置文件中，params部分是特定于主题的; 其内容可能因主题而异。

    我们用YAML元数据语言编写了清单2.5中的配置文件，我们将在第3章中讨论。 它使用由冒号分隔的键和值提供结构化信息。 YAML是人类可读且区分大小写的，但空白的变化可能会导致YAML解析器出现问题。

    Hugo还支持更 “间距友好” 的TOML格式。 本书的参考资料还包含配置文件的TOML版本。 如果你使用它作为替代，则应删除config.yaml。

    注 在发布之前更新网站的实际基本URL，而不是配置文件中的http://example.org/。 离开Example.org文件会破坏网站中的绝对链接。

    Eclectic的主题允许我们提供我们的徽标，甚至通过将这些放置在资产/图像文件夹中 (而不是在主题/Eclectic/资产/图像文件夹中) 来控制网站背景图像。 我们将在此文件夹中放置logo.svg和Backack.svg以个性化网站。 如果不存在此文件夹，我们将需要创建它。 (你可能需要重新启动开发服务器才能使更改生效。) 这些文件存在于第2章的代码包中 ( https://github.com/hugoinaction/hugoinaction/tree/chapter-02-resources/03)。

    2.3.2 内容页面 #

    网站的目标是提供内容，到目前为止，我们美丽的网站上还没有内容。 在网站顶部菜单添加的条目链接到不存在的页面！ 我们需要在网站上创建页面以使其正常运行。 我们将首先将内容添加到本章菜单中链接到的页面，然后在第三章中设置内容格式。

    我们可以将内容页面创建为内容文件夹中的文本或标记文件。 我们可以将一个带有基于Markdown的内容的Priacy.md文件放在该文件夹中，以获得https：//Localhost：1313/Privacy URL。 同样，我们可以添加关于.md，credits.md，terms.md和contact.md页面 ( https://github.com/hugoinaction/hugoinaction/ tree/chapter-02-resources/04)。 Hugo会自动应用主题，一旦添加了文档，页面就会呈现出来。 这样，我们可以添加任意数量的页面，以生成网站的核心结构。 Markdown提供了各种格式选项，我们将在第三章中研究这些选项。

    2.3.3 索引页 #

    索引页(也称为主页或登录页)是网站的第一页，负责引导用户了解期望的内容。 它的内容是独特的，与所有其他页面不同。 基于文本的内容适用于某些页面，但许多网站实现了索引页的自定义内容。 网站甚至有量身定制的旋转木马和带有大量图像的部分，需要自定义实现。 Hugo认识到了这一点，并为索引页提供了一个唯一的模板，称为索引模板。 在许多主题中，索引模板是以特定于主题的方式自定义的，并且索引页面配置不能跨主题移植。

    注 大多数Hugo主题都提供了一个名为ExampleSite的文件夹，其中包含一个使用该主题的入门网站。 此文件夹在浏览特定于主题的配置和自定义选项时非常有用。

    Hugo的模板是HTML文件，但这些模板可以是任何基于文本的文件格式 (例如，JSON，XML，甚至纯文本)，并带有参与编译步骤的附加模板标记。 对于试图构建自定义Hugo模板的用户来说，最好从索引模板开始，因为它只影响网站的一个页面。 可以使用layouts文件夹覆盖Hugo模板。 在本章中，我们将不使用任何模板标记，而是从一个普通的HTML模板开始，我们将把该模板放置为layout/index.html。 它仍然是Hugo模板，可以访问所有变量，这些变量是可选的。

    对于Acme Corporation的索引页，我们将用一个定制页面覆盖主题的索引页，该页面用HTML和CSS硬编码，如图2.6所示。 此页面将包含网站徽标，标题，副标题，带有号召性用语的按钮 (告诉读者进行更多探索) 以及带有指向其他页面的链接的页脚。

    在Layout文件夹中，我们将放置一个名为index.html的新文件，其中包含自定义的HTMLContent( https://github.com/hugoinaction/hugoinaction/tree/chapter-02-资源/05)。 因为我们不使用Hugo的模板语言，所以我们将对所有路径进行硬编码，并使用到各种支持托管位置的相对位置。

    图2.6我们可以在Hugo网站中创建自定义登录页面，方法是将一个名为index.html的文件放在Layout文件夹中。 此页面覆盖主题提供的主页。 对于Acme Corporation，我们使用了一个带有硬编码的HTML和CSS的登录页面，并避开了ececectic的特定于主题的特性来创建基于结构化数据的页面。

    如清单2.6所示，我们可以通过在layouts文件夹中放置HTML模板文件来覆盖Hugo主题中的模板。 这样做为理解HTML的人提供了一种快速定制网站的方法，而不需要学习Hugo。 自定义HTML对于特定网站可以是唯一的。 除非我们使用Hugo的模板语言，否则我们必须小心使用我们正在编写的HTML，因为定制的HTML页面不会在它链接的内容更改时自动更改。

    Listing 2.6 Overriding Hugo’s themes (layouts/index.html) 
    <!DOCTYPE html>
<html lang="en">
<head>
<meta charset="UTF-8">
<meta name="description" content="Welcome to the website of Acme Corporation, the leading creator
of digital shapes on the planet, providing precise shape creations that are ready to use.">
<meta name="viewport" content="width=device-width,
initial-scale=1.0" />
<link rel="stylesheet" href="./index.css">
<title>Acme Corporation</title>
</head>
<body class="home">

资源的相对路径。当我们在托管环境中使用子文件夹发布此代码时，绝对路径会导致问题。

<section>
<img src="./image/logo.svg" alt="Acme Logo" width="64"/>
<h1>Acme Corporation</h1>
<h2>Shaping the world for you to live in</h2>
<a href="./blog">Explore</a>
</section>
<footer>

静态文件夹中的资源。 HTML中引用的资产应在静态文件夹中提供，以确保最终网站中的正确链接。

<a href="./about">About Us</a>
<a href="./privacy">Privacy Policy</a>
<a href="./terms">Terms of Use</a>
<a href="./contact">Contact Us</a>
</footer>
</body>
</html>

硬编码菜单。 在纯HTML中，我们必须假设菜单项的url及其名称与指定的内容匹配。
    注 Hugo不修改模板内部提供的HTML。

    普通的HTML文件需要图像和索引文件才能正常工作。 我们为Eclectic主题放置的资产文件夹中的图像需要使用Hugo Pipes。 (我们将在第6章讨论Hugo管道。)对于不需要处理的内容，我们必须使用静态文件夹。 在我们开始使用Hugo的资产处理管道之前，我们需要在静态文件夹中放置资产的第二个副本。 这包括静态/index.css、静态/Image/ackather.svg、静态/Image/logo.svg和静态/Favic.ico。 这些资产随章节资源 (https:// github.com/hugoinaction/hugoinaction/tree/ chapter-02-resources/06) 一起提供。
